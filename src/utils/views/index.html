Bienvenido al backend de huertos

Con este back-end podras registrar y obtener información sobre tus huertos. Estos son sus diferentes end-points . Además debes de tener en cuenta que cuando veas un ? en alguna de las variables solicitadas ya sea en el body o en los parametros, quiere decir que puedes omitir ese valor o colocar el valor, ya que es una variable opcional.

End points para los usuarios - /api/v1/user
POST /api/v1/user/register - Con este end point podras registrarte como usuario.
BODY

{

"nick": "pepito",

"password": "pepito123"

}

POST /api/v1/user/login - Con este end point podras ingresar como usuario de la plataforma.
BODY

{

"nick": "pepito",

"password": "pepito123"

}

POST /api/v1/user/register-huerto - Con este end point podras registrar los huertos de un usuario.
HEAD

{

"Autorization": "bearer token-authentication"

}

BODY

{

"name": "bearer token-authentication"

}

GET /api/v1/user/huertos/:limit/:page - Con este end point podras listar los diferentes huertos de un usuario.
HEAD

{

"Autorization": "bearer token-authentication"

}

End points para los arduinos - /api/v1/arduino
POST /api/v1/arduino/register-climate - Con este end-point registraras el clima del huerto que estes diagnosticando o necesitas verificar el comportamiento climatico del mismo.
HEAD

{

"Autorization": "bearer token-authentication"

}

BODY

{

"id_huerto": "1b9d6bcd-bbfd-4b2d-9b5d-ab8dfbbd4bed",

"temperatura": 1.25,

"humedad": 0.25

}

GET /api/v1/arduino/climates/:id/:limit/:page - Con este end point podras listar el log de cambio de clima de cada uno de los huertos.
HEAD

{

"Autorization": "bearer token-authentication"

}

End points para el historial de plantas - /api/v1/plant
POST /api/v1/plant/register - Con este end-point registraras una planta al historial informativo de plantas.
BODY

{

"name_plant": "Arbol de mango",

"description": "Es un arbol de mangos",

"clime": "Climas humedos y calidos",

"flora": "Cada 2 meses da flor",

"height": 10.26,

"humidity": [10, 10],

"light": 10.26,

"pests": ["Pajaros", "Hormigas"],

"solutions": ["Para evitar los pajaros en los arboles...", "Para evitar las hormigas..."],

"space_earth": 10.5,

"specific_care": "Algo",

"type_earth": "Tierra humedad",

"type_fruit": "Mango"

}

POST /api/v1/plant/edit/:id - Con este end-point actualizaras la información de una planta. Además todas las variables dentro del body son OPCIONALES.
BODY

{

"name_plant": "Arbol de mango",

"description": "Es un arbol de mangos",

"clime": "Climas humedos y calidos",

"flora": "Cada 2 meses da flor",

"height": 10.26,

"humidity": [10, 10],

"light": 10.26,

"pests": ["Pajaros", "Hormigas"],

"solutions": ["Para evitar los pajaros en los arboles...", "Para evitar las hormigas..."],

"space_earth": 10.5,

"specific_care": "Algo",

"type_earth": "Tierra humedad",

"type_fruit": "Mango"

}

GET /api/v1/plant/list/:limit/:page/:name? - Con este end point podras listar el log de cambio de clima de cada uno de los huertos.
GET /api/v1/plant/plant/:id - Con este end point podras obtener una planta en especifico usando el id de la planta.
DELETE /api/v1/plant/plant/:id - Con este end-point podras eliminar una planta en especifico.

| Tabla de códigos de respuesta |

Esta es la tabla de los códigos de respuesta que encontraras en cada response de cada end-point, el código lo podras extraer de la key http_code.

código
descripción
5000

Este código hace referencia a un error interno en los microservicios.

5001

Este código hace referencia a un error interno al momento de hacer un registro en la base de datos.

4001

Este código hace referencia a un error de validación de los valores envíados en la petición, ya que estos puedan que no correspondan a los valores solicitados.

4002

Este código hace referencia a un error en la validación al momento de enviar la respuesta, especificamete quiere decir que un usuario o huerto o objeto ya existe con el valor unico ya sea nombre o nick.

4003

Este código hace referencia a un error, donde define que el objeto, usuario, huerto no existe aún en la base de datos.

4004

Este código hace referencia al error NOT FOUND, que quiere decir que no encontró los resultados buscados (Es especial para busquedas en especifico o para listar.).

2000

Este código hace referencia a que la petición se a completado correctamente y se entrego una respuesta satisfactoria.

2001

Este código hace referencia a un registro de datos completado correctamente.
